How does it work

  • In order to guarantee test standardization and equality all tests are done using docker containers.

  • A container is created and then the module is tested inside it, saving it's exit codes. In the end the container is killed and removed.

  • If your module was updated since last test it will be tested again.

  • Test scripts like: "'echo "Error: no test specified" && exit 1" are considered dummy test scripts.
